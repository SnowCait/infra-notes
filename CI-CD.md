# CI/CD

## なぜ CI/CD をするのか
### Unit Test, Feature Test
- コードにバグがないか常に監視できる

### CD
- 手動でやるとヒューマンエラーが発生する
- コスト削減
  - 手動でやるとコストが高い

### 全般
- コードとして表せる
  - バージョン管理できる
- ログが残る
  - 後で調査ができる
